    Authentication

    APIs for handling authenticating users

    Authenticate a user by email and password

    For use if the user's rememver token is no longer valid.

    Example request:

    curl -X POST \
    "http://localhost/api/login" \
    -H "Content-Type: application/json" \
    -H "Accept: application/json" \
    -d '{"email":"tim01@smith.com","password":"password1234"}'

    const url = new URL(
    "http://localhost/api/login"
);

let headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "tim01@smith.com",
    "password": "password1234"
}

fetch(url, {
    method: "POST",
    headers: headers,
    body: body
})
    .then(response => response.json())
    .then(json => console.log(json));

    Example response (200):

    {
    "date": 1591997000,
    "status": "OK",
    "message": "",
    "data": {
        "token": "147WecYkPYL5LppPIg5m5LY5d9NBoBUyn6Z65lPnuwfiahY4B86zCmcKFx6S0sJoSz3TCSrDNOCmjPZn",

    HTTP Request

    POST api/register

    Body Parameters

    Parameter Type Status Description
    email string required The email of the account to be created.
    password string required The password of the account to be created.
    password_confirmation string required Repeat of the password of the account to be created.
    first_name string required First name of user.
    last_name string required Last name of user.
    address_line_1 string required First line of address of user.
    address_line_2 string optional Second line of address of user.
    city string required City of user.
    state string required Two digit state abbreviation of US state of user.
    country string required Country of user.
    referer string required Entity who referred the user to Ameelio Letters.
    postal string required Zip code of user.
    phone string required Phone number of user.
    s3_img_url string optional AWS S3 URL of the profile picture of user.
